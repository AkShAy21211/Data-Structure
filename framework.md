## The Thinking Framework for Every Problem
#### Before writing a single line of code, ask yourself these questions in order:

- What is the input and output? Write it down explicitly.
- What constraints matter? Size of input? Sorted? Duplicates allowed?
- What pattern does this look like? Subarray? Pair? Traversal?
- What's the brute force? Always think of it — it clarifies the problem.
- Can I optimize it? What would reduce the time complexity?


## What "Done" Actually Means for a Problem
#### A problem is only done when you can:

- Explain the approach out loud in plain English before coding
- Code it from scratch without reference
- State the time and space complexity confidently
- Solve a slight variation of it (interviewer follow-up)

If you can't do all four — it's not done, it's just seen.
